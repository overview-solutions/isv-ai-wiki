Village metering

ISV Tech Comm primary endeavor · 2026

Mini-grid operators need to bill each home, shut off non-payers, and catch stolen power — without buying a city-scale metering system. This section documents affordable ways to do that: what problems matter, which meter features to ask for, and how to connect different brands through a pole-mounted gateway.

Why IEEE · why ISV

Rural metering stalls when every vendor ships a closed bundle, every tender copies utility software requirements, and operators cannot swap gear without rewriting billing. Meter makers, integrators, open-platform builders, and field NGOs all have legitimate interests — but no single company can credibly write the common data contract everyone else should adopt. IEEE Smart Village sits outside those product lines: a neutral place to compare field evidence and publish draft specs without picking a winner.

Goal: more integrators bidding on the same scope, mixed meter brands on one feeder, and migration paths when a vendor exits — gateways compete on implementation, not proprietary lock-in.

How data moves (simple view)

Each meter brand uses its own wire protocol under the pole. The edge gateway (pole box or data concentrator) translates to one northbound report. Prepaid tokens are a payment path — SMS, mobile money, keypad — separate from scheduled meter reads.

Home meter → Radio / wire to pole → Pole gateway → Standard JSON report → Billing · mobile money

Why village tenders go wrong

Utility scope on a village budget

Consultants paste full billing-suite and 15-minute load-profile requirements into a 200-meter tender. Appropriate for a city — not for a $40–110/connection feeder.

Mixing unlike things

Prepaid tokens, meter reads, inverter monitoring, and cloud format are different layers. One RFP line that says “all protocols” creates impossible bids. See Problems diagram.

Closed vendor bundles

Mesh + cloud + billing from one vendor ships fast but blocks second-source hardware and migration.

Comms fragility

When the pole uplink fails, customers still need tokens and keypads must still work. May 2026 notes on redundant paths.
