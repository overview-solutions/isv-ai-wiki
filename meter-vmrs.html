OBIS · VMRS

v0.1-draft · Jun 2026 · NOT a published standard

Before citing in an RFP VMRS is an ISV working checklist — not IEC/DLMS UA/STS. Primary artifact: northbound-mqtt-v0.1.json. VMRS appendix: vmrs-registers.json. Validate tier B against vendor ICD (Interface Class Description).

Northbound JSON/MQTT at the gateway is the interoperability contract. VMRS names which OBIS keys should appear in that payload — mostly tier A codes every DLMS meter already has.

Tier A · IEC OBIS1.8.0, 32.7.0, etc. Universal — cite in JSON, don’t reinvent.
Tier B · ICD variesCredit, disconnect, serial, tamper — need vendor ICD excerpt per model.
Tier C · optionalLast-token audit — operator need, weak OBIS standardization.

Northbound JSON example

{
  "schemaVersion": "0.1",
  "ts": "2026-06-22T14:30:00Z",
  "feederId": "feeder-north-paravur-01",
  "gatewayId": "pole-ems-0042",
  "meterSerial": "DDZ1513-88421",
  "registers": [
    { "obis": "1.8.0", "value": 1842.5, "unit": "kWh", "quality": "good" },

POC playbook · SIP

POC ask: one feeder, 10–30 meters, northbound-mqtt-v0.1 on schedule, STS vending unchanged, ICD gap matrix published. Ask vendor for DLMS/AMI engineer — not sales@.

SIP (Glenn Algie): Session Information Protocol for authenticated disconnect/market sessions above MQTT — not VoIP SIP, not SunSpec.
